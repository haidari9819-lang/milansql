Why MilanSQL?

Everything a database needs.
Nothing it doesn't.

Built phase-by-phase from the ground up — every feature is hand-crafted in pure C++17 without a single external library.

🔒

ACID Transactions

Full MVCC with Write-Ahead Logging, SAVEPOINT, ROLLBACK TO SAVEPOINT, and SELECT FOR UPDATE row-level locking — battle-tested durability.

Learn more →
⚡

Zero Dependencies

Pure C++17 standard library only. No Boost, no OpenSSL, no third-party headers. Clone, compile, run — on Linux, Windows, or macOS.

View source →

5 Network Protocols

TCP(4406) + MySQL Wire(4407) + PostgreSQL Wire(5433) + REST/HTTP(8080) + GraphQL(8081). Connect with psql, mysql CLI, curl, or any HTTP client.

Code Examples

SQL you already know.
Features you didn't expect.

MilanSQL speaks standard SQL — and adds table inheritance, column stores, pub/sub, and more. 

-- Phase 78: Table Inheritance
CREATE TABLE person (id INT PRIMARY KEY, name TEXT);
CREATE TABLE employee (salary REAL) INHERITS (person);

INSERT INTO person   VALUES (1, 'Alice');
INSERT INTO employee VALUES (2, 'Bob', 75000);

SELECT * FROM person;          -- returns Alice + Bob (inherited)
SELECT * FROM ONLY person;     -- returns only Alice

-- Phase 75: Row-Level Security
ENABLE ROW LEVEL SECURITY ON employee;
CREATE POLICY own_rows ON employee
  FOR SELECT TO alice
  USING (id = CURRENT_USER_ID());

-- Phase 82: Adaptive Optimizer
SET QUERY_REWRITE = ON;
EXPLAIN REWRITTEN SELECT * FROM employee WHERE 1=1;
-- Rewrite: removed always-true condition WHERE 1=1

SHOW INDEX SUGGESTIONS;
-- CREATE INDEX idx_employee_salary ON employee(salary)  -- 78%

-- Phase 76: LISTEN / NOTIFY pub/sub
LISTEN updates;
NOTIFY updates, 'new_employee_added';
# DB-API 2.0 compatible Python client (stdlib only)
import milansql

# Connect via DSN (Phase 79)
conn = milansql.connect("milansql://root:pass@localhost:4406/mydb")
cursor = conn.cursor()

# Execute SQL
cursor.execute("""
    CREATE TABLE orders (
        id    INT PRIMARY KEY AUTO_INCREMENT,
        item  TEXT NOT NULL,
        price REAL
    )
""")

cursor.execute("INSERT INTO orders VALUES (NULL, 'Laptop', 999.99)")
conn.commit()

# Fetch results
cursor.execute("SELECT * FROM orders WHERE price > %s", (100,))
rows = cursor.fetchall()
for row in rows:
    print(row)

conn.close()
# Start REST API server
$ ./build/milansql.exe --http --port 8080

# Execute a query via HTTP POST
$ curl -s -X POST http://localhost:8080/query \
  -H 'Content-Type: application/json' \
  -d '{"sql": "SELECT * FROM orders LIMIT 10"}'

  "columns": ["id", "item", "price"],

  "elapsed_ms": 0.42
}

Getting Started

Up and running
in three steps.

No package manager, no Docker, no config files. Just a C++17 compiler and CMake.

1

Clone & Build

git clone https://github.com/
  haidari9819-lang/milansql
cd milansql
cmake -B build -G Ninja
cmake --build build
2

Start the Server

# HTTP + Web Dashboard
./build/milansql.exe \
  --http --port 8080

# TCP Server
./build/milansql.exe \
  --server --port 4406

# Interactive REPL
./build/milansql.exe
3

Connect & Query

# Via REST API
curl -X POST \
  http://localhost:8080/query \
  -H 'Content-Type: application/json'\
  -d '{"sql":"SHOW TABLES"}'

# Via DSN (Python)
import milansql
conn = milansql.connect(
  "milansql://root@localhost/db"
)
