Capabilities
Everything a database needs

From SQL parsing to network replication — every component is implemented from scratch.

⚡

Zero Dependencies

Pure C++17 standard library only. No Boost, no OpenSSL, no third-party headers. Compiles on Linux, macOS, and Windows with any C++17 compiler.

🔒

ACID Transactions

WAL-based crash recovery, BEGIN/COMMIT/ROLLBACK, SAVEPOINT, row-level SELECT FOR UPDATE locks, and table-level LOCK TABLE READ|WRITE.

🌐

TCP/IP + REST API

Multi-threaded TCP server on port 4406 with connection pooling. HTTP/JSON REST API on port 8080 with Web Dashboard. Multi-statement query support.

📊

Query Optimizer

Cost-based optimizer selects optimal join order and index access paths. EXPLAIN/EXPLAIN ANALYZE with real execution times. LRU/TTL query cache.

🔄

Master/Slave Replication

Binary log with position tracking. Slave polls master every 500ms, auto-reconnects on failure. Read-only slave enforcement. SHOW MASTER/SLAVE STATUS.

📅

Event Scheduler

CREATE EVENT … ON SCHEDULE EVERY n DAY|HOUR|MINUTE. One-time and recurring jobs. Persisted to database.events. SET EVENT_SCHEDULER = ON|OFF.

🔍

Full-Text Search

CREATE FULLTEXT INDEX, MATCH(col) AGAINST('term') with TF-ranking. Multi-column full-text indexes.

🗂

Stored Procedures + Cursors

DECLARE CURSOR FOR SELECT, OPEN/FETCH/CLOSE, CONTINUE HANDLER FOR NOT FOUND, LOOP/LEAVE, IF/ELSE/END IF — full procedural SQL.

📦

Partitioning

RANGE, LIST, and HASH partitioning. Partition pruning in query optimizer. ALTER TABLE ADD/DROP PARTITION. SHOW PARTITIONS.

🪟

Window Functions

ROW_NUMBER(), RANK(), DENSE_RANK(), SUM/AVG/COUNT/MIN/MAX OVER (PARTITION BY … ORDER BY …).

📋

INFORMATION_SCHEMA

Full virtual schema: tables, columns, indexes, views, triggers, routines, partitions, events, user_privileges.

💾

Backup / CSV

BACKUP DATABASE TO 'file.sql' (mysqldump-compatible), RESTORE FROM, LOAD DATA INFILE, SELECT … INTO OUTFILE. RFC-4180 CSV support.

Get Started
Up and running in 60 seconds

Clone, build, and execute your first queries with no configuration needed.

terminal — build 
# Clone the repository
git clone https://github.com/haidari9819-lang/milansql
cd milansql

# Build (Linux / macOS)
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build

# Windows (MSYS2 UCRT64)
cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build

# Run REPL
./build/milansql

# Run as TCP server
./build/milansql --server --port 4406

# Run as HTTP/REST server
./build/milansql --http --port 8080
milansql> — first queries 
milansql> CREATE TABLE users (
    id    INT PRIMARY KEY AUTO_INCREMENT,
    name  TEXT NOT NULL,
    score INT DEFAULT 0
  )
  Tabelle 'users' erstellt.

milansql> INSERT INTO users VALUES
    (NULL, 'Alice', 95), (NULL, 'Bob', 82)
  2 Zeile(n) eingefuegt.

milansql> SELECT name, score
    FROM users
    WHERE score > 80
    ORDER BY score DESC
