Features

⚡

Zero Dependencies

Pure C++17 with no external libraries. Runs anywhere a C++ compiler exists — Linux, Windows, macOS.

🔒

ACID Transactions

Full BEGIN/COMMIT/ROLLBACK with WAL (Write-Ahead Logging), SAVEPOINT, and SELECT FOR UPDATE row locking.

🌐

TCP/IP + REST API

Built-in TCP server (port 4406) and HTTP/JSON REST API (port 8080) with Web Dashboard.

📊

Query Optimizer

Cost-based optimizer with join reordering, index selection, EXPLAIN ANALYZE, and query result caching.

🔄

Master/Slave Replication

Binary log replication with automatic sync, auto-reconnect, and read-only slave enforcement.

📅

Event Scheduler

MySQL-compatible CREATE EVENT with EVERY n SECOND/MINUTE/HOUR/DAY schedules and AT timestamp.

🔍

Full-Text Search

FULLTEXT INDEX with MATCH ... AGAINST and TF-based relevance scoring across multiple columns.

📦

Partitioning

Table partitioning by RANGE, LIST, and HASH with automatic partition pruning in queries.

🧮

Generated Columns

GENERATED ALWAYS AS (expr) STORED/VIRTUAL — computed columns with arithmetic expressions, auto-updated on INSERT/UPDATE.

🪟

Window Functions

ROW_NUMBER(), RANK(), DENSE_RANK(), SUM/AVG/COUNT OVER (PARTITION BY … ORDER BY …).

🗂

Stored Procedures + Cursors

DECLARE CURSOR, OPEN/FETCH/CLOSE, CONTINUE HANDLER FOR NOT FOUND, LOOP/LEAVE, IF/ELSE.

🐍

Python + Node.js Clients

DB-API 2.0 compatible Python client and Node.js HTTP client — zero dependencies, stdlib only.

🔐

Row-Level Security

ENABLE/DISABLE RLS per table, CREATE POLICY with USING expressions, CURRENT_USER_ID() predicate substitution.

📡

LISTEN / NOTIFY Pub/Sub

Channel-based pub/sub messaging with payload support, LISTEN/UNLISTEN/NOTIFY commands, thread-safe delivery.

⚙️

Parallel Query Execution

std::thread-based parallelFilter and parallelAggregate, configurable threshold and worker count, PARALLEL(N) hint.

🧬

Table Inheritance

PostgreSQL-style CREATE TABLE child INHERITS (parent), inherited SELECT, FROM ONLY, SHOW INHERITANCE tree.

📈

Column Store Engine (OLAP)

CREATE COLUMN TABLE for OLAP workloads — columnar SUM/AVG/COUNT/MIN/MAX scans, GROUP BY fallback, SHOW STORAGE FORMAT.

🔗

Logical Replication

CREATE PUBLICATION FOR TABLE, CREATE SUBSCRIPTION CONNECTION, append-only change log, ALTER SUBSCRIPTION ENABLE/DISABLE.

🧠

Adaptive Query Optimizer

Automatic query rewriting (1=1 removal, redundant predicates), adaptive index suggestions from query patterns, EXPLAIN REWRITTEN.

🔌

Connection Strings (DSN)

milansql://user:pass@host:port/db, mysql://, jdbc:milansql:// — auto-connect and USE on startup.

Quick Start

# Clone and build
git clone https://github.com/haidari9819-lang/milansql
cd milansql && cmake -B build && cmake --build build

# Start REPL
./build/milansql.exe

# Start TCP Server
./build/milansql.exe --server --port 4406

# Start REST API + Web Dashboard
./build/milansql.exe --http --port 8080
-- Generated Columns (Phase 68)
CREATE TABLE produkte (
  id    INT  PRIMARY KEY AUTO_INCREMENT,
  name  TEXT NOT NULL,
  netto REAL,
  brutto REAL GENERATED ALWAYS AS (netto * 1.19) STORED,
  mwst   REAL GENERATED ALWAYS AS (netto * 0.19) VIRTUAL
);

INSERT INTO produkte VALUES (NULL, 'Laptop', 1000);
INSERT INTO produkte VALUES (NULL, 'Maus',   25);

SELECT * FROM produkte;
-- brutto: 1190 / 29.75  |  mwst: 190 / 4.75  (auto-computed)

-- Transactions + Savepoints
BEGIN;
  UPDATE produkte SET netto = 800 WHERE name = 'Laptop';
  SAVEPOINT after_update;
  UPDATE produkte SET netto = 0 WHERE name = 'Maus';
  ROLLBACK TO SAVEPOINT after_update;
COMMIT;
