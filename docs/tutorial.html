Getting Started Tutorial

From zero to advanced SQL in 8 steps. No external dependencies required — just a C++17 compiler and CMake.

1

Installation & Build

Clone the repository and build with CMake + Ninja. MilanSQL has zero external dependencies.

bash (Linux / macOS)
git clone https://github.com/haidari9819-lang/milansql
cd milansql
cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build
./build/milansql
bash (Windows — MSYS2 UCRT64)
export PATH="/c/msys64/ucrt64/bin:$PATH"
cmake -B build -G Ninja
cmake --build build
./build/milansql.exe
Tip: Run ./build/milansql_tests.exe to verify all 1221 tests pass.
2

First Queries

Start the interactive REPL and create your first table. MilanSQL persists all data to database.milan automatically.

SQL
CREATE TABLE employees (
  id     INT  PRIMARY KEY AUTO_INCREMENT,
  name   TEXT NOT NULL,
  dept   TEXT,
  salary REAL DEFAULT 0
);

INSERT INTO employees (name, dept, salary) VALUES
  ('Alice', 'Engineering', 95000),
  ('Bob',   'Engineering', 88000),
  ('Carol', 'Marketing',   72000);

SELECT * FROM employees;
SELECT dept, AVG(salary) FROM employees GROUP BY dept;
Tip: Use \dt to list tables, \d tablename to describe, \q to quit.
3

JOINs & Subqueries

MilanSQL supports INNER, LEFT, RIGHT, and FULL OUTER JOINs as well as correlated subqueries.

SQL
CREATE TABLE departments (id INT PRIMARY KEY, name TEXT, budget REAL);
INSERT INTO departments VALUES (1, 'Engineering', 500000), (2, 'Marketing', 200000);

-- INNER JOIN
SELECT e.name, e.salary, d.budget
FROM employees e INNER JOIN departments d ON e.dept = d.name;

-- IN subquery
SELECT * FROM employees
WHERE dept IN (SELECT name FROM departments WHERE budget > 300000);

-- Correlated subquery
SELECT name, salary,
  (SELECT AVG(salary) FROM employees e2 WHERE e2.dept = e.dept) AS dept_avg
FROM employees e;
4

Transactions & ACID

MVCC + Write-Ahead Log for full ACID guarantees. Supports nested savepoints and SELECT FOR UPDATE row locking.

SQL
BEGIN;
UPDATE employees SET salary = salary * 1.1 WHERE dept = 'Engineering';
COMMIT;

-- Savepoints
BEGIN;
INSERT INTO employees (name, dept, salary) VALUES ('Frank', 'HR', 55000);
SAVEPOINT after_frank;
INSERT INTO employees (name, dept, salary) VALUES ('Grace', 'HR', 52000);
ROLLBACK TO SAVEPOINT after_frank;  -- undo Grace, keep Frank
COMMIT;

Advanced Features

Window functions, Recursive CTEs, Table Inheritance, and Array operations.

SQL — Window Functions
SELECT name, dept, salary,
  RANK()      OVER (PARTITION BY dept ORDER BY salary DESC) AS rank,
  SUM(salary) OVER (PARTITION BY dept) AS dept_total
FROM employees;
SQL — Recursive CTE
WITH RECURSIVE counter(n) AS (
  SELECT 1
  UNION ALL
  SELECT n + 1 FROM counter WHERE n < 10
)
SELECT * FROM counter;
SQL — Table Inheritance
CREATE TABLE person  (id INT PRIMARY KEY, name TEXT);
CREATE TABLE employee (salary REAL) INHERITS (person);

INSERT INTO person   (id, name) VALUES (1, 'Alice');
INSERT INTO employee (id, name, salary) VALUES (2, 'Bob', 75000);

SELECT * FROM person;       -- Alice + Bob (2 rows)
SELECT * FROM ONLY person;  -- Alice only (1 row)

Performance & Query Planning

EXPLAIN, ANALYZE, adaptive optimizer, column store for OLAP, and parallel query execution.

SQL
CREATE INDEX idx_emp_salary ON employees(salary);
EXPLAIN SELECT * FROM employees WHERE salary > 80000;
ANALYZE TABLE employees;
SHOW STATISTICS FOR employees;

SET QUERY_REWRITE = ON;
EXPLAIN REWRITTEN SELECT * FROM employees WHERE 1=1;
SHOW INDEX SUGGESTIONS;

SET MAX_PARALLEL_WORKERS = 4;
SELECT /*+ PARALLEL(4) */ COUNT(*) FROM employees;

Client Drivers

Connect from Python, Node.js, Java (JDBC), Rust, .NET, psql, or any HTTP client.

bash — Start servers
./build/milansql.exe --http --http-port 8080 --pg --pg-port 5433 --graphql --graphql-port 8081
Python — DB-API 2.0
import milansql
conn = milansql.connect("milansql://root@localhost:4406/mydb")
cur = conn.cursor()
cur.execute("SELECT name, salary FROM employees WHERE salary > %s", (80000,))
print(cur.fetchall())
curl — REST API
curl -X POST http://localhost:8080/query \
  -H 'Content-Type: application/json' \
  -d '{"sql":"SELECT * FROM employees LIMIT 5"}'
GraphQL Playground: Open http://localhost:8081/graphql/playground in your browser.

Replication & Pub/Sub

Logical replication between instances, LISTEN/NOTIFY pub/sub, and Change Data Capture.

SQL — LISTEN / NOTIFY
LISTEN employee_changes;
NOTIFY employee_changes, 'new_hire:Alice';
UNLISTEN *;
SQL — Logical Replication
CREATE PUBLICATION mypub FOR TABLE employees, departments;
CREATE SUBSCRIPTION mysub
  CONNECTION 'milansql://root@primary:4406/db'
  PUBLICATION mypub;
SHOW LOGICAL LOG;
SQL — Change Data Capture
ALTER TABLE employees ENABLE CDC;
SELECT * FROM cdc.employees;
SELECT * FROM cdc.employees AFTER SEQUENCE 3;
